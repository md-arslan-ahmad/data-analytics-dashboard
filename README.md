# Data Analyst · Data Engineering · Fullstack Assignment

## Project Structure

```
.
├── clean_data.py          # Part 1 — Data cleaning
├── analyze.py             # Part 2 — Merging & analysis
├── generate_data.py       # Helper to generate sample CSVs
├── Dockerfile             # Docker image for backend
├── docker-compose.yml     # Docker Compose (backend + frontend)
├── backend/
│   ├── main.py            # FastAPI REST API
│   └── requirements.txt
├── frontend/
│   └── index.html         # Single-page React dashboard
├── data/
│   ├── raw/               # Original CSVs
│   └── processed/         # Output CSVs (generated by scripts)
└── tests/
    └── test_cleaning.py   # Pytest unit tests (bonus)
```

---

## Prerequisites

- Python 3.9+
- pip
- Node is NOT required (frontend is vanilla HTML + CDN React)

---

## Setup

### 1. Install Python dependencies

```bash
pip install pandas numpy fastapi uvicorn pydantic pytest
```

### 2. Generate sample data (if raw CSVs not provided)

```bash
python generate_data.py
```

This creates `data/raw/customers.csv`, `data/raw/orders.csv`, `data/raw/products.csv`.

---

## Running the Pipeline

### Step 1 — Clean the data

```bash
python clean_data.py
# Optional flags:
# --raw-dir data/raw     (default)
# --out-dir data/processed  (default)
```

Outputs:
- `data/processed/customers_clean.csv`
- `data/processed/orders_clean.csv`

### Step 2 — Merge & Analyse

```bash
python analyze.py
# Optional flags:
# --processed-dir data/processed   (default)
# --raw-dir data/raw               (default)
# --out-dir data/processed         (default)
```

Outputs:
- `data/processed/monthly_revenue.csv`
- `data/processed/top_customers.csv`
- `data/processed/category_performance.csv`
- `data/processed/regional_analysis.csv`

### Step 3 — Start the Backend API

```bash
uvicorn backend.main:app --reload --port 8000
```

API will be available at `http://localhost:8000`.

Endpoints:
| Endpoint | Description |
|---|---|
| `GET /health` | Health check |
| `GET /api/revenue` | Monthly revenue trend |
| `GET /api/top-customers` | Top 10 customers + churn flag |
| `GET /api/categories` | Category performance |
| `GET /api/regions` | Regional KPIs |

### Step 4 — Open the Frontend

Open `frontend/index.html` in your browser directly, or serve it:

```bash
# Python simple server
cd frontend && python -m http.server 3000
# Then open http://localhost:3000
```

---

## Running with Docker (Bonus)

```bash
# Build and start both backend and frontend
docker-compose up --build

# Backend API: http://localhost:8000
# Frontend:    http://localhost:3000
```

To update processed CSVs without rebuilding:
```bash
python clean_data.py && python analyze.py
# CSVs in data/processed/ are bind-mounted into the container
```

## Running Tests (Bonus)

```bash
pytest tests/ -v
```

Tests cover:
- `is_valid_email` — valid/invalid/null cases
- `parse_date_multi` — three date formats + unparseable
- `normalize_status` — variant mapping + null handling
- `clean_customers` — deduplication, email lowercasing, whitespace stripping
- `clean_orders` — unrecoverable row dropping, status normalization, date parsing

## Assumptions Made

1. **Date ambiguity**: For `DD/MM/YYYY` vs `MM/DD/YYYY`, the assignment specifies `DD/MM/YYYY` so dates like `01/06/2023` are treated as 1 June, not 6 January.
2. **Status normalization**: Any status not in `{completed, pending, cancelled, refunded}` after mapping is logged as a warning and kept as-is rather than silently dropped.
3. **Churn definition**: "No completed orders in the past 90 days" is computed relative to the **maximum order_date in the dataset**, not today's wall-clock date, for reproducibility.
4. **Product join**: Orders are matched to products by `orders.product == products.product_name` (exact match, case-sensitive). Unmatched orders retain `NaN` for product columns but are not dropped.
5. **Median fill for amount**: Median is computed per-product across the entire dataset (not per month), so it is stable regardless of row ordering.
6. **Sample data**: Since CSVs were not provided, realistic synthetic data is generated by `generate_data.py`. The same cleaning/analysis pipeline works identically on real data placed in `data/raw/`.
